# 🛠️ Project Design Documentation  

This repository contains the detailed **design plan** for the project, broken down into three core tasks. Each task represents a major component of the design process, providing structured documentation, diagrams, and architectural decisions essential for development and future maintenance.

---

## Task 1: **🔄 System Architecture Design**  
This task focuses on the high-level architecture and infrastructure of the system, including the relationships between core components. 

### **Components:**
- **Architecture Diagram**: Visualizes the client-server model and the database interactions.
- **Deployment Diagram**: Shows the distribution of components across servers and clients.
- **Technology Stack**: Lists the frameworks, programming languages, and services used.

**📂 Files:**
- `vpp/architecture_diagram.vpp`
- Exported Diagram: `diagrams/architecture_overview.png`  
- Documentation: `docs/system_architecture.md`

---

## Task 2: **📊 UML Design and Workflow Diagrams**  
This task includes detailed UML diagrams that describe the structure and behavior of the system, ensuring all functionalities are well-documented.

### **Components:**
- **Class Diagram**: Defines the key entities and relationships between them.
- **Sequence Diagram**: Illustrates the interaction flow between components during key processes (e.g., ticket purchase).
- **Activity Diagram**: Outlines workflows such as refunds and complaint handling.

**📂 Files:**
- `vpp/uml_design.vpp`
- Exports: `diagrams/class_diagram.png`, `diagrams/sequence_diagram.pdf`, `diagrams/activity_diagram.png`
- Documentation: `docs/uml_workflows.md`

---

## Task 3: **📑 Requirements, Use Cases, and Decision Logs**  
This task provides a comprehensive breakdown of the system's requirements and user interactions through use cases, along with key design decisions taken throughout the project.

### **Components:**
- **Requirements Document**: Details the functional and non-functional requirements.
- **Use Case Diagrams**: Visualizes user interactions with the system.
- **Decision Log**: Tracks important design and technology choices.

**📂 Files:**
- `vpp/use_case_diagram.vpp`
- Exports: `diagrams/use_case.png`
- Documentation: `docs/requirements_and_decisions.md`
